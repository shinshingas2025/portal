GetTopLevelMessages 預存程序

說明：

此預存程序會傳回入口網站內特定 Discussion 模組中每個執行緒的第一項訊息。輸入參數為 ModuleID 與 PortalID。

計算的欄位 ChildCount 含有對最上層訊息回覆的總數。如果此數字大於 1，Discussion 模組便會將訊息轉換為可展開的節點。

定義： 

    
   CREATE PROCEDURE Portal_GetTopLevelMessages
   (
       @ModuleID int
   )
   AS
   
   SELECT
       ItemID,
       DisplayOrder,
       LEFT(DisplayOrder, 23) AS Parent,    
       (SELECT COUNT(*) -1  FROM Portal_Discussion Disc2 WHERE LEFT(Disc2.DisplayOrder,LEN(RTRIM(Disc.DisplayOrder))) = Disc.DisplayOrder) AS ChildCount,
       Title,  
       CreatedByUser,
       CreatedDate
   
   FROM Portal_Discussion Disc
   
   WHERE 
       ModuleID=@ModuleID
     AND
       (LEN( DisplayOrder ) / 23 ) = 1
   
   ORDER BY
       DisplayOrder
使用的資料庫資料表：

Discussion：如 Discussion Portal 模組所示，Discussion 資料表中的每項記錄都是執行緒討論中的訊息。由於所有的Discussion 模組都將其記錄存放在此資料表中，因此每個項目都包含 ModuleID 以容許在單一查詢中擷取相關項目。

在 Discussion 資料表中，DisplayOrder 欄位是一項相當有趣的功能，可以用來建立討論中訊息的執行緒顯示。此欄位的計算方式，是將代表目前日期的 23 個字元字串，串連至項目的父項目 DisplayOrder 所得來的。這樣一來，訊息即可透過簡單的遞增排序，以正確的順序顯示。這個做法會因 DisplayOrder 欄位大小而有其固有限制。此例中有 750 個字元，表示在巢狀結構中 32 層以下的訊息，都無法以正確的順序顯示。

此資料表的主索引鍵為 ItemID 識別欄位。請注意訊息本文不得超過 3000 個字元。
