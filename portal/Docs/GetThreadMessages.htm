GetThreadMessages 預存程序

說明：

此預存程序傳回 ItemID 指定的最上層訊息的所有子項計數。

此預存程序包含一項簡單功能，幫助 Discussion 模組在轉換執行緒時，顯示巢狀化的息。GetThreadMessages 以每項訊息巢狀階層的比例為準，以數個 HTML 空白字元填滿 Indent 欄位。Discussion 模組會在Title 欄位前直接轉換 Indent 欄位以建立巢狀化的視覺效果。

定義： 

            
   CREATE PROCEDURE Portal_GetThreadMessages
   (
       @Parent nvarchar(750)
   )
   AS
   
   SELECT
       ItemID,
       DisplayOrder,
       REPLICATE( ' ', ( ( LEN( DisplayOrder ) / 23 ) - 1 ) * 5 ) AS Indent,
       Title,  
       CreatedByUser,
       CreatedDate,
       Body
   
   FROM Portal_Discussion
   
   WHERE
       LEFT(DisplayOrder, 23) = @Parent
     AND
       (LEN( DisplayOrder ) / 23 ) > 1
   
   ORDER BY
       DisplayOrder
使用的資料庫資料表：

Discussion：如 Discussion Portal 模組所示，Discussion 資料表中的每項記錄都是執行緒討論中的訊息。由於所有的Discussion 模組都將其記錄存放在此資料表中，因此每個項目都包含 ModuleID 以容許在單一查詢中擷取相關項目。

在 Discussion 資料表中，DisplayOrder 欄位是一項相當有趣的功能，可以用來建立討論中訊息的執行緒顯示。此欄位的計算方式，是將代表目前日期的 23 個字元字串，串連至項目的父項目 DisplayOrder 所得來的。這樣一來，訊息即可透過簡單的遞增排序，以正確的順序顯示。這個做法會因 DisplayOrder 欄位大小而有其固有限制。此例中有 750 個字元，表示在巢狀結構中 32 層以下的訊息，都無法以正確的順序顯示。

此資料表的主索引鍵為 ItemID 識別欄位。請注意訊息本文不得超過 3000 個字元。
