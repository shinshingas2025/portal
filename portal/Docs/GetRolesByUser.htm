GetRolesByUser 預存程序

說明：

此預存程序傳回指定使用者所屬的入口網站安全性角色。輸入參數為使用者的電子郵件地址。

定義： 
    
   CREATE PROCEDURE Portal_GetRolesByUser
   (
       @Email         nvarchar(100)
   )
   AS
   
   SELECT  
       Portal_Roles.RoleName,
       Portal_Roles.RoleID
   
   FROM Portal_UserRoles
     INNER JOIN 
       Portal_Users ON Portal_UserRoles.UserID = Portal_Users.UserID
     INNER JOIN 
       Portal_Roles ON Portal_UserRoles.RoleID = Portal_Roles.RoleID
   
   WHERE   
       Portal_Users.Email = @Email
使用的資料庫資料表：

UserRoles：UserRoles 資料表提供入口網站安全性角色 (定義於 Roles 資料表) 與使用者 (定義於 Users 資料表) 之間的多對多連接。在使用 UserRoles 資料表時，每名使用者都能屬於多種角色，而且每項角色的成員也可以為多名使用者。

UserRoles 資料表沒有主索引鍵。

Users：Users 資料表中的每項記錄都是唯一使用者識別項。此資料表的主索引鍵為 UserID 識別欄位。

Roles：Roles 資料表中的每項記錄都在選取的入口中定義一項唯一角色。並透過 UserRoles 查詢值資料表，將使用者加入至角色中。

此資料表的主索引鍵為 RoleID 識別欄位。
